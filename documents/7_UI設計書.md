# UI設計書

## 概要

本ドキュメントは、MonsterWalker（dqwfunモンスターウォーカー）のUI/UX設計を記載しています。React Native + Expoで開発されるモバイルアプリのUI仕様、画面構成、デザイン方針を定義します。

---

## デザイン方針

### 基本コンセプト
- **シンプルで直感的なUI**: 攻略情報の検索・登録が迷わず行える設計
- **情報の視認性**: パーティ構成や行動順序が一目でわかる表示
- **片手操作**: スマートフォンでの片手操作を考慮したボタン配置
- **5タブナビゲーション**: ホーム/お気に入り/登録/ランキング/マイページ

### デザイン原則
1. **シンプル**: 必要な情報に素早くアクセス可能
2. **わかりやすい**: 視覚的に理解しやすいアイコンと色使い
3. **効率的**: 少ないタップで目的の操作を完了
4. **快適**: ストレスのない応答速度とアニメーション

- **iOS**: San Francisco（システムデフォルト）
- **Android**: Roboto（システムデフォルト）

### フォントサイズ
- **特大**: 24px - 画面タイトル

- **Bold (700)**: 見出し、ボタン
- **Medium (500)**: 強調テキスト
- **Regular (400)**: 本文、通常テキスト

## 画面階層構造

### 初回起動時の画面遷移

```
スプラッシュ画面 (A01) → ホーム画面 (H01)（未ログイン状態）
                                ↓
                      ログイン必須機能を実行
                                ↓
                        ログイン画面 (A02) へ遷移
```

## まとめ

MonsterWalkerのUI設計は以下の特徴があります：

- **シンプルで直感的**: 攻略情報の検索・登録が迷わず行える設計
- **情報の視認性**: パーティ構成、役割、行動順序が一目でわかる
- **5タブナビゲーション**: ホーム/お気に入り/登録/ランキング/マイページ
- **片手操作対応**: スマートフォンでの片手操作を考慮したボタン配置
- **iOS/Android対応**: クロスプラットフォーム対応のUI設計

この設計により、ユーザーは効率的にモンスター攻略情報を検索・登録・共有することができます。
